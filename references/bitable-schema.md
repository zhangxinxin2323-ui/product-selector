# 飞书表结构契约

所有 base token 和 table_id 从 `config.local.json` 或环境变量读取。本文只定义逻辑表和字段，不保存正式凭据。

## 共同字段

所有可写表建议具备：

| 字段 | 类型 | 用途 |
|---|---|---|
| 分析键 | Text | 幂等 upsert |
| 首次分析日期 | DateTime | 首次创建保留 |
| 最近分析日期 | DateTime | 每次更新 |
| 数据站点 | Select | US/GB/DE/JP 等 |
| 证据索引 | Text/URL | 指向本地或云端 evidence index |
| 报告版本 | Number | 更新递增 |

## finance

- 关联初选 / 初筛 / 开发
- 分析键、模型版本、决策策略版本、输入哈希、计算模式
- 售价、产品成本、头程、落地成本
- FBA费、佣金率、仓储费、退货率
- 当前 CPC、当前点击 CVR、广告订单占比
- 净利润、净利率、M值、广告空间、盈亏平衡CPC、盈亏平衡CVR
- Financial Decision、Launch Feasibility
- 最大可承受落地成本、目标落地成本上限、隐含采购价上限
- 首批数量、首批投入、峰值资金需求、最大损失估算
- 回本月份、首次月利润转正月份
- 12月总广告费、12月累计利润、期末现金流
- 悲观情景峰值资金、悲观情景回本月份
- 财务置信度、假设与置信度、证据索引

估算成本不得写入真实采购价字段。
目标落地成本上限表示“采购 + 单件头程”的合计门槛。没有可追溯头程估算时，隐含采购价上限留空。
百分比字段写入前读取 Base 字段格式，按 `fraction` 或 `whole` 生成；不得靠字段名猜测缩放。公式和查找引用字段不直接写入。

## ai_market_analysis — 后验追踪字段

运营复盘时填写（手动）。用于发现"什么样的数据画像可以成功"的规律。

| 字段名 | 类型 | 说明 | 何时填 |
|--------|------|------|--------|
| 最终决策 | Select | 实际进入/实际放弃/观望中 | 决策后 |
| 实际上线时间 | DateTime | 产品实际上线日期 | 上线后 |
| 6个月后状态 | Select | 成功(盈利)/平(保本)/失败(亏损)/未满6月 | 上线6月后 |
| 决策复盘 | Select | 决策正确/决策错误/不确定 | 回顾时 |
| 关键成败因素 | Text | 回头看最重要的1-2个成功/失败原因 | 回顾时 |

**用法**：积累 20+ 条记录后用飞书视图交叉筛选。
例：筛选"成功"的记录 → 发现 M>2.0 且 CPC<$1.0 且新品率>15% 同时出现 → 规律。
