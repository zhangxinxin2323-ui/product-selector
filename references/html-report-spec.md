# HTML 决策卡规范

> HTML 只做决策卡（≤2屏），不再输出完整 HTML 报告。完整分析看 MD。

## 结构（6 区块）

| 区块 | 内容 | 必选 |
|------|------|:--:|
| Header | 决策标签(GO=绿/CONDITIONAL=橙/NO-GO=红) + 细分市场名 + 参考ASIN/品牌 + 日期 + 大号评分 | ⛔ |
| KPI 网格 | 8 卡片(2行×4列)：搜索量/CPC/毛利率/新品率/Top3集中度/评论门槛/均价/趋势方向 | ⛔ |
| 关键发现 | 3-5 bullet，绿色=优势，红色=风险，灰色=中性 | ⛔ |
| 风险与缓解 | 2-3 条，高风险(红)+中风险(橙)，每条含缓解方案 | ⛔ |
| 下一步行动 | 依赖用户输入的下一步 | ⛔ |
| Footer | 数据来源+日期+飞书链接+完整报告文件名 | ⛔ |

## 颜色体系

```css
--go: #16a34a;      /* 绿色 — GO / 优秀指标 */
--orange: #ea580c;  /* 橙色 — CONDITIONAL / 警告 */
--red: #dc2626;     /* 红色 — NO-GO / HOLD / 风险 */
```

| 条件 | 颜色 |
|------|:--:|
| 评分 ≥ 7.5 | 绿 GO |
| 6.0 ≤ 评分 < 7.5 | 橙 CONDITIONAL |
| 评分 < 6.0 | 红 NO-GO / HOLD |
| 新品率 ≥ 15% | 绿 |
| 评论门槛 < 300 | 绿 |
| CPC < $1 | 绿 |
| 新品率 < 10% / 评论 > 1000 | 红 |

## 响应式

- 桌面：KPI 2 行 × 4 列
- 手机 (≤500px)：KPI 4 行 × 2 列
- 风险卡片：桌面 2 列，手机 1 列

## 文件命名

```
~/product-selector-reports/<市场名或关键词>-<YYYYMMDD>/decision-card.html
```
