## 快速开始

### 最简单的用法——给一个 ASIN

```
帮我分析 B0F9JZSK2Q
```

AI 自动执行：产品详情 → 关键词分析 → 类目 Top100 → 属性标注 → 三层决策 → 写入飞书。

### 给一个关键词/品类名

```
printer stand 这个品类值得做吗
```

AI 从关键词出发定位类目，然后执行完整分析。

### 中途给成本数据

```
成本 $3.2，头程 $0.8，CPC $0.85，CVR 9%
```

AI 收到后自动触发财务分析，计算 M 值、净利率、回本周期等 48 个字段。

## 核心概念：三层决策

> ⚠️ **最重要**：Market（市场好不好）≠ Financial（能不能赚钱）≠ Launch（资金撑不撑得住）

```
           Market Decision（市场可行性）
                  ↓
           Financial Decision（财务可行性）
                  ↓
           Launch Feasibility（资金可行性）
                  ↓
           Overall Decision（综合决策）
```

**任何一层失败，整体不得为 GO。** 这个设计防止"市场好但亏钱"的错误判断（如Aquarium Stands：AI评分8.05但M=1.02亏损）。

### 三层决策与 AI-推荐推进的对应关系

| Market | Financial | Launch | Hard Gates | AI-推荐推进 |
|:------:|:--------:|:------:|:----------:|-----------|
| GO (≥7.5) | GO (M≥2.0, 净利率≥15%) | GO/通过 | 全部 pass | **GO-推荐进入** |
| GO (≥7.5) | COND (M≥1.6, 净利率≥10%) | PENDING | pass/pending | **CONDITIONAL GO-有条件** |
| COND (6.0-7.4) | GO (M≥2.0) | PENDING | pass/pending | **CONDITIONAL GO-有条件** |
| COND (6.0-7.4) | HOLD (M≥1.3) | PENDING | pass | **HOLD-暂缓** |
| 任意 | NO-GO (M<1.3或亏本) | 任意 | 任意 | **NO-GO-放弃** |
| 任意 | 任意 | 任意 | fail | **NO-GO-放弃** |

## 入口路径：ASIN vs 关键词

### 路径 A：从 ASIN 进入（完整路径）

```
ASIN (如 B0F9JZSK2Q)
    │
    ├─→ ProductRequest → 精确售价/FBA/佣金/利润率
    ├─→ BsrCategory → 精准 nodeId
    ├─→ ASINRequestKeywordv2 → 反查核心关键词
    │
    ↓ 数据更精确、信息更全
    ↓ 可以做精确的财务分析
    ↓
    全流程：初选 → AI分析 → 初筛 → 财务 → 开发 → 报告
```

**特点**：可以做精确正向财务（P&L 48字段），因为售价/FBA/佣金全部来自 ProductRequest 实测。

### 路径 B：从关键词进入（市场级）

```
关键词 (如 printer stand)
    │
    ├─→ KeywordRequest → 搜索量/CPC/关联类目
    ├─→ CategoryRequest(精准nodeId) → Top100
    ├─→ 从 Top100 选取参考 ASIN
    │
    ↓ 市场级分析，缺参考 ASIN 的精确售价/FBA
    ↓ 可以做反向财务（成本上限）
    ↓
    初选 → AI分析 ─→ GO/COND → 初筛 → 等用户给ASIN → 财务
                  ─→ HOLD/NO-GO → 停止
```

**特点**：只能做反向财务（"M≥2.0 需要采购价≤$X"），无法做精确 P&L。适合市场初筛。

### 两条路径的差异

| 维度 | ASIN 入口 | 关键词入口 |
|------|:--:|:--:|
| 市场分析 | ✅ 完整 | ✅ 完整 |
| 属性标注 | ✅ | ✅ |
| Go/No-Go | ✅ 精确评价 | ✅ 市场级 |
| **精确财务（正向 P&L）** | ✅ 售价/FBA/佣金实测 | ❌ 需用户提供参考 ASIN |
| **反向财务（成本上限）** | ✅ | ✅ "M≥2.0 需采购≤$X" |
| 初筛 | ✅ GO/COND触发 | ✅ GO/COND触发 |
| 产品开发 | ✅ | ✅（细分市场级，不依赖特定ASIN） |

## 完整流程路径图

```
用户输入（ASIN 或 关键词）
    │
    ▼
Step 1: 市场发现
    ├─ 产品详情 / 关键词分析
    ├─ 类目定位（nodeId）
    ├─ Top100 产品 + 趋势
    ├─ 至少 3 个维度的关键词
    ├─ 差评采集（VOC）
    │
    ├─→ 写入「产品初选」表
    ├─→ 写入「AI分析参考」表（29+字段）
    │
    ├─→ ⛔ 弹出运营数据输入窗口（成本弹窗）
    │      用户可选：现在填 / 跳过 / 飞书手动填
    │
    ▼
Step 2: 市场可行性（Market Decision）
    ├─ score_go_nogo.py 计算 Market 评分
    ├─ hard gates 检查（专利/合规/安全/供应链）
    │
    ▼  ┌─────────────────────────────────────┐
    │  │ GO (≥7.5) / CONDITIONAL (≥6.0)      │
    │  │   → 继续 Step 3                      │
    │  │                                       │
    │  │ HOLD (4.0-5.9) / NO-GO (<4.0)        │
    │  │   → 停止。输出报告。记录决策依据。    │
    │  └─────────────────────────────────────┘
    │
    ▼
Step 3: 属性标注 + 交叉分析 + VOC
    ├─ attribute-tagger.py 自动标注 Top100
    ├─ 交叉矩阵 → 识别空白/薄供给
    ├─ VOC 差评按维度归类 → 痛点→方案映射
    │
    ├─→ GO/CONDITIONAL → 自动写入「产品初筛」表（11/16 AI填写）
    ├─→ GO/CONDITIONAL → 自动写入「产品开发」表（12/18 AI填写）
    │
    ▼
Step 4: 财务可行性（Financial Decision）
    ├─ 条件：AI分析参考表中 1688采购价 + 头程 + CPC + CVR 全有值
    │
    ├─ 缺成本 → finance.py reverse → 输出采购价上限
    ├─ 有成本 → finance.py forward → 48字段 P&L
    │
    ├─→ Financial GO → 写入「财务分析」表
    ├─→ Financial PENDING/NO-GO → 不写入
    │
    ▼
Step 5: Launch Feasibility（资金可行性）
    ├─ 条件：可用资金 + 最大回本期限 + MOQ
    ├─ dynamic.js → 3情景（乐观/基准/悲观）
    ├─ 峰值资金需求 / 回本月份 / 悲观情景压力
    │
    ▼
Step 6: 输出交付
    ├─ decision-card.html（速判卡，手机可看）
    ├─ full-report.md（≥10章完整报告）
    ├─ report_lint.py 检查通过
    └─ ⛔ 流程完整性检查表（16项核对）
```

## 6 张飞书表的推进逻辑

| # | 表名 | table_id | 写入条件 | 写入内容 |
|---|------|----------|---------|---------|
| 1 | **产品初选** | tbljJBWDxaLerJsN | **始终写入** | ASIN/关键词/类目/价格带/推进状态 |
| 2 | **AI分析参考** | tblMeN1P5LvXsp2e | **始终写入** | 29+字段（23结构化+8文本+4运营） |
| 3 | **产品初筛** | tblPhc8dHOelKPPF | GO 或 CONDITIONAL GO | 11/16 AI填写 + 5待人工确认 |
| 4 | **财务分析** | tblllu3JJYrG5KGE | **仅 Financial GO** | 48字段 P&L + 模型版本 + 输入哈希 |
| 5 | **产品开发** | tbl3dtXNC3Emy38k | GO 或 CONDITIONAL GO | 12/18 AI填写 + 成本待确认 |
| 6 | **供应商管理** | tblu1Q70vnmsiBZ5 | 用户提供1688链接后 | 供应商名/链接/价格/MOQ/样品 |

### 重要规则

- **AI分析参考每次必写**，无论后续是否推进。它是数据沉淀的核心表。
- **财务分析仅在 Financial GO 时写入**。CONDITIONAL/HOLD/NO-GO 不写估算数据。
- **产品初筛和产品开发并行触发**，都依赖同一个条件（GO/CONDITIONAL），不是串行依赖。
- **NO-GO/HOLD 时后续表不写入是正确行为**，不是流程遗漏。

## 所有关键字段阈值

### Market Decision（市场可行性）

| 维度 | 权重 | 评分标准 | 数据来源 |
|------|:---:|---------|---------|
| **市场规模** | 25% | 搜索量>100K→9-10, 50-100K→7-8, 20-50K→5-6, <20K→3-4 | KeywordRequest.SearchVolume |
| **竞争格局** | 31.25% | ClickShare<20%→蓝海9-10, 20-40%→7-8, 40-60%→5-6, >60%→3-4 | KeywordRequest + CategoryRequest |
| **需求清晰度** | 18.75% | CVR≥5%+VOC明确→9-10, 2-5%→7-8, <2%→5-6 | KeywordRequest + ProductReviewsQuery |
| **进入壁垒** | 25% | Review<300+新品>15%→低壁垒8-10, Review500-1000→中壁垒5-7, Review>1000→高壁垒3-4 | CategoryRequest |

| 市场分数 | 决策 | 含义 |
|:---:|------|------|
| ≥7.5 | Market GO | 市场结构优秀 |
| 6.0-7.4 | Market CONDITIONAL | 市场可进入但需注意 |
| 4.0-5.9 | Market HOLD | 市场有结构性缺陷 |
| <4.0 | Market NO-GO | 不建议进入 |

### Financial Decision（财务可行性）— decision.js v1.0.0

| 指标 | GO | CONDITIONAL GO | HOLD | NO-GO |
|------|:--:|:--:|:--:|:--:|
| **M值** | ≥ 2.0 | ≥ 1.6 | ≥ 1.3 | < 1.3 |
| **净利率** | ≥ 15% | ≥ 10% | ≥ 0% | < 0%（亏损） |

| M值 | 含义 |
|:--:|------|
| >2.5 | 🟢 优秀 — 每投入$1收回>$2.5 |
| 2.0-2.5 | 🟢 健康 — 投入产出比良好 |
| 1.6-2.0 | 🟡 可控 — 可以放量但需关注 |
| 1.3-1.6 | 🟡 危险 — 利润安全垫薄 |
| 1.0-1.3 | 🔴 极度危险 — 勉强回本 |
| <1.0 | 🔴 亏损 — 每卖一件亏一件 |

### Hard Gates（一票否决项）

| Gate | fail条件 | 影响 |
|------|---------|------|
| **专利** | 核心功能被有效专利保护且无法规避 | fail → 整体NO-GO |
| **合规** | 需要强制认证（FDA/FCC等）且无法获取 | fail → 整体NO-GO |
| **产品安全** | 存在已知安全隐患 | fail → 整体NO-GO |
| **供应链** | MOQ过高或供应商不可得 | fail → 整体NO-GO |

pending → 整体最多 CONDITIONAL GO

### 竞争指标阈值

| 指标 | 🟢 蓝海 | 🟡 正常 | 🟠 注意 | 🔴 危险 |
|------|:--:|:--:|:--:|:--:|
| **Top3点击集中度** | <20% | 20-40% | 40-60% | >60% |
| **新品率(≤6月)** | >20% | 15-20% | 10-15% | <10% |
| **平均评论门槛** | <300 | 300-500 | 500-1000 | >1000 |
| **CPC** | <$1.0 | $1.0-2.0 | $2.0-3.0 | >$3.0 |
| **品牌数** | >80 | 50-80 | 20-50 | <20 |
| **FBA占比** | <70% | 70-90% | 90-97% | >97% |

### 搜索趋势方向判定

| 方向 | 条件 | 含义 |
|------|------|------|
| 快速增长 | 近12月增长率>20%持续 | 市场扩张期，最佳入场时机 |
| 稳定增长 | 近12月增长5-20% | 健康增长，可进入 |
| 平稳 | 增长率-5%~5% | 成熟市场 |
| 增速放缓 | 曾经高增长但近期增幅收窄 | 市场趋于饱和 |
| 下降 | 持续负增长 | 市场萎缩，不推荐 |

### 新品友好度

| 评级 | 新品占比 | 含义 |
|------|:--:|------|
| 友好 | ≥15% | 新品牌有机会进入并站稳 |
| 中等 | 10-15% | 可以进入，需要差异化 |
| 困难 | 5-10% | 新品突围难度较大 |
| 非常困难 | <5% | 几乎无新品能进入Top100 |

## 运营数据输入（必读）

### 四个必填字段

| 字段 | 从哪来 | 为什么重要 | Sorftime能提供吗？ |
|------|--------|----------|:--:|
| **1688采购价** | 1688搜索/供应商报价 | 决定M值的核心变量 | ❌ |
| **头程运费** | 货代报价 | 决定落地成本 | ❌ |
| **当前CPC** | 广告后台 Campaign实际数据 | 决定广告成本 | 🟡 市场CPC可参考KeywordRequest.Cpc |
| **当前点击CVR** | **广告后台** — 点击→购买转化率 | 决定adCPA | ❌ CLI+MCP共11端点均不提供 |

> ⚠️ **Serftime 的 SearchConversionRate = 搜索→购买转化率，不是点击 CVR！** 用它算广告成本会得到错误结果。点击 CVR 必须从广告后台获取。

### 三种输入方式

| 方式 | 操作 | AI动作 |
|------|------|--------|
| A | 对话中告诉AI："成本$3.2 头程$0.8 CPC$0.85 CVR 9%" | AI写入飞书 → 自动触发财务分析 |
| B | 在飞书 AI分析参考表中手动填写 | 下次对话开始AI回读发现有值 → 自动触发 |
| C | 分批给：先给成本，后面再给CPC/CVR | 缺哪个补哪个，补齐自动触发 |

### 广告订单占比（可选，默认50%）

这是你运营判断的值——多少比例订单来自广告。精铺通常30-50%。可随时调整。

## 常见问题

### Q1: 为什么分析完了没有初筛/财务/开发？
A: 评分 NO-GO 或 HOLD 时，后续表不写入是正确行为。检查「AI-推荐推进」字段——只有 GO/CONDITIONAL GO 才触发后续。

### Q2: 我要怎么做才能触发财务分析？
A: 在 AI分析参考表中填入4个字段：1688采购价 + 头程运费 + 当前CPC + 当前点击CVR。或者在对话中说"成本$X 头程$Y CPC$Z CVR W%"

### Q3: 搜索转化率能当CVR用吗？
A: **不能。** SearchConversionRate（搜索→购买）≠ 点击CVR（点击→购买）。用错了会导致M值算偏高——可能漏判"亏损"产品。

### Q4: CPC为什么有两个字段？
A: 核心词CPC（市场参考值，来自KeywordRequest）vs 当前CPC（你的实际出价，来自广告后台）。财务分析用后者。

### Q5: 属性标注为什么有70% unknown？
A: 标题正则解析覆盖率约30-70%。需调 product_detail 补充。品类维度越标准，覆盖率越高。

### Q6: 分析一次大概消耗多少API？
A: 快速模式约25次，完整模式约60次，深度VOC另加。查看余额：`sorftime whoami`。

### Q7: 从关键词进入和从ASIN进入有什么区别？
A: 见[入口路径章节](#入口路径asin-vs-关键词)。核心区别：ASIN能做精确财务（正向P&L），关键词只能做反向财务（成本上限）。
